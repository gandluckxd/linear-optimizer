# Консольный запуск оптимизации MOS

`main.py` больше не создаёт окно PyQt. Он запускает полный рабочий поток для
одного `GRORDERS_MOS_ID`:

1. получает связанные СЗ и данные материалов через существующий API;
2. запускает неизменённые линейный и плоскостной оптимизаторы;
3. записывает `OPTIMIZED_MOS` и `OPTDETAIL_MOS`;
4. корректирует материалы и распределяет ячейки.

## Конфигурация

Рядом с EXE должен находиться файл `optimizer.config.json`. Шаблон с прежними
значениями интерфейса находится в этой папке. Все параметры линейной и
плоскостной оптимизации вынесены в него. В `api.base_url` укажите адрес
запущенного API-сервиса, который имеет доступ к Firebird.

Параметры `operations.adjust_materials` и `operations.distribute_cells` по
умолчанию включены. Отключать их допустимо только для диагностического прогона:
в production-потоке все три этапа записи должны выполняться.

## Запуск

```bat
linear-optimizer-headless.exe 12345
```

Здесь `12345` — `GRORDERS_MOS_ID`, а не обычный `ORDERID`: существующая схема
по этому идентификатору находит связанные СЗ через `GRORDER_UF_VALUES`.

Для проверки входных данных и алгоритмов без изменения Firebird:

```bat
linear-optimizer-headless.exe 12345 --dry-run
```

Другой путь конфигурации можно передать явно:

```bat
linear-optimizer-headless.exe 12345 --config D:\Altawin\optimizer.config.json
```

При успешном запуске процесс завершается с кодом `0`; при ошибке — с кодом
`1`, а причина выводится в журнал.

## Сборка EXE

Выполнить из папки `client` в подготовленном Python-окружении:

```bat
pyinstaller --onefile --name linear-optimizer-headless main.py
copy optimizer.config.json dist\optimizer.config.json
```

Конфиг намеренно остаётся внешним файлом: его можно менять без пересборки EXE.
